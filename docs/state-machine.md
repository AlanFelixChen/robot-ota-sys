# OTA系统状态机设计文档

## 1. 概述

### 1.1 文档目的
定义机器人端OTA升级过程中的**完整状态机模型**，明确各状态的定义、转移条件及异常处理路径，为OTA守护进程的开发与测试提供依据。

### 1.2 状态机范围
覆盖从升级指令接收到升级完成（或回滚）的全生命周期，包括Linux主控固件升级和MCU协同升级。

---

## 2. 状态定义

### 2.1 状态总览

```
                    ┌──────────┐
                    │   IDLE   │ ←────────────────────────────┐
                    └────┬─────┘                              │
                         │ 收到升级指令                         │
                    ┌────▼─────┐                              │
                    │ PRECHECK │                              │
                    └────┬─────┘                              │
                         │ 预检通过                            │
                    ┌────▼──────┐                             │
                    │ DOWNLOADING│←────────┐                  │
                    └────┬──────┘         │ 断点续传          │
                         │ 下载完成        │                  │
                    ┌────▼─────┐          │                  │
                    │ VERIFYING│          │                  │
                    └────┬─────┘          │                  │
                         │ 校验通过        │                  │
                    ┌────▼─────┐          │                  │
                    │ FLASHING │          │                  │
                    └────┬─────┘          │                  │
                         │ 写入完成        │                  │
                    ┌────▼──────┐         │                  │
                    │ ACTIVATING│         │                  │
                    └────┬──────┘         │                  │
                         │ 重启完成        │                  │
                    ┌────▼──────┐         │                  │
                    │ COMMITTING│─────────┼─── 健康检查失败 ──┐
                    └────┬──────┘         │                  │
                         │ 确认成功        │                  │
                         └────────────────┘                  │
                                                             │
                    ┌──────────┐                             │
                    │ ROLLBACK │←────────────────────────────┘
                    └────┬─────┘
                         │ 回滚完成
                         └──────────→ IDLE

                    ┌──────────┐
                    │  ERROR   │←── 任意阶段不可恢复错误
                    └──────────┘
```

### 2.2 状态详细定义

| 状态 | 状态码 | 说明 | 允许动作 |
|------|--------|------|---------|
| IDLE | 0 | 空闲状态，等待升级指令 | 接收升级指令、查询版本 |
| PRECHECK | 1 | 升级前预检，检查升级条件 | 存储检查、电量检查、网络检查 |
| DOWNLOADING | 2 | 正在下载固件包 | 断点续传、进度上报、暂停/取消 |
| VERIFYING | 3 | 校验固件签名与完整性 | ECDSA验签、SHA256校验 |
| FLASHING | 4 | 将固件写入备用分区 | 分区写入、差分还原、进度上报 |
| ACTIVATING | 5 | 设置启动标记，准备重启 | 设置U-Boot环境变量、重启设备 |
| COMMITTING | 6 | 新系统启动后健康检查 | 健康检查、版本确认、COMMIT标记 |
| ROLLBACK | 7 | 回滚至旧版本 | 分区切换、状态上报 |
| ERROR | 8 | 不可恢复错误 | 错误上报、等待云端指令 |

## 3. 状态转移条件

### 3.1 正常流程转移

| 源状态 | 目标状态 | 转移条件 | 触发事件 |
|--------|---------|---------|---------|
| IDLE | PRECHECK | 收到云端MQTT升级指令 | `ota_upgrade_cmd` |
| PRECHECK | DOWNLOADING | 存储空间≥固件大小×1.5、电量≥30%或充电中、网络延迟<500ms | `precheck_pass` |
| DOWNLOADING | VERIFYING | 固件包下载完成，字节数匹配 | `download_complete` |
| VERIFYING | FLASHING | ECDSA签名验证通过，SHA256校验通过 | `verify_pass` |
| FLASHING | ACTIVATING | 固件写入备用分区完成，分区校验通过 | `flash_complete` |
| ACTIVATING | COMMITTING | 设备重启完成，新系统启动成功 | `boot_new_partition` |
| COMMITTING | IDLE | 健康检查通过，COMMIT标记写入 | `commit_success` |

### 3.2 异常转移

| 源状态 | 目标状态 | 转移条件 | 触发事件 |
|--------|---------|---------|---------|
| PRECHECK | IDLE | 预检条件不满足（空间/电量/网络） | `precheck_fail` |
| DOWNLOADING | DOWNLOADING | 下载中断，支持断点续传 | `download_interrupted` |
| DOWNLOADING | ERROR | 下载重试超过3次仍失败 | `download_retry_exhausted` |
| VERIFYING | ERROR | 签名校验失败或SHA256不匹配 | `verify_fail` |
| FLASHING | ERROR | Flash写入失败，重试3次仍失败 | `flash_write_fail` |
| COMMITTING | ROLLBACK | 健康检查失败或超时未确认 | `health_check_fail` |
| ROLLBACK | IDLE | 回滚完成，旧系统正常运行 | `rollback_complete` |
| 任意状态 | ERROR | 发生不可恢复的系统异常 | `fatal_error` |

### 3.3 外部指令转移

| 源状态 | 目标状态 | 转移条件 | 触发事件 |
|--------|---------|---------|---------|
| DOWNLOADING | IDLE | 云端下发取消升级指令 | `ota_cancel_cmd` |
| 任意状态 | ROLLBACK | 云端下发强制回滚指令 | `ota_rollback_cmd` |
| ERROR | IDLE | 云端下发错误清除指令 | `ota_clear_error_cmd` |

## 4. 各状态内部处理逻辑

### 4.1 IDLE 状态

```
进入动作：
  - 重置所有升级相关变量
  - 上报当前状态至云端

循环动作：
  - 监听MQTT升级指令Topic
  - 响应版本查询请求

退出动作：
  - 记录升级开始时间戳
  - 初始化升级上下文（目标版本、固件URL、校验信息）
```

### 4.2 PRECHECK 状态

```
进入动作：
  - 上报状态：PRECHECK

检查项：
  1. 存储空间检查：
     - 获取备用分区可用空间
     - 条件：可用空间 ≥ 固件包大小 × 1.5
     - 失败原因：STORAGE_INSUFFICIENT
  
  2. 电池电量检查：
     - 读取电源MCU电量数据
     - 条件：电量 ≥ 30% 或 充电状态 = true
     - 失败原因：BATTERY_LOW
  
  3. 网络质量检查：
     - Ping云端服务器，获取RTT
     - 条件：RTT < 500ms，丢包率 < 10%
     - 失败原因：NETWORK_POOR
  
  4. 版本兼容性检查：
     - 校验目标版本与当前硬件兼容
     - 校验MCU固件版本兼容矩阵
     - 失败原因：VERSION_INCOMPATIBLE

退出动作：
  - 上报预检结果（通过/失败及原因）
```

### 4.3 DOWNLOADING 状态

```
进入动作：
  - 上报状态：DOWNLOADING
  - 初始化下载进度计数器

循环动作：
  - 使用HTTPS Range请求下载固件块
  - 每下载1MB上报一次进度
  - 支持暂停/取消指令

断点续传逻辑：
  - 记录已下载字节偏移量
  - 中断恢复时从偏移量继续下载
  - 临时文件存储在数据分区

退出动作：
  - 校验下载文件大小与预期一致
  - 上报下载完成状态
```

### 4.4 VERIFYING 状态

```
进入动作：
  - 上报状态：VERIFYING

校验流程：
  1. SHA256完整性校验：
     - 计算下载固件的SHA256哈希
     - 与云端下发的哈希值比对
     - 失败：VERIFY_HASH_MISMATCH
  
  2. ECDSA签名校验：
     - 从ATECC608读取公钥
     - 验证固件签名的有效性
     - 失败：VERIFY_SIGNATURE_INVALID
  
  3. 版本信息校验：
     - 解析固件头，提取版本号
     - 与升级指令中的目标版本比对
     - 失败：VERIFY_VERSION_MISMATCH

退出动作：
  - 上报校验结果
```

### 4.5 FLASHING 状态

```
进入动作：
  - 上报状态：FLASHING
  - 确定目标分区（当前App A运行 → 写入App B）

写入流程：
  1. 差分还原（如为差分包）：
     - 读取当前分区固件
     - 与差分包合并生成完整新固件
     - 校验还原后固件的完整性
  
  2. 擦除目标分区：
     - 擦除App B分区全部块
     - 验证擦除完成
  
  3. 写入新固件：
     - 分块写入，每块写入后校验
     - 写入失败自动重试（最多3次）
     - 每写入10%上报进度

退出动作：
  - 校验写入分区与源固件一致
  - 上报写入完成状态
```

### 4.6 ACTIVATING 状态

```
进入动作：
  - 上报状态：ACTIVATING
  - 禁止新的升级指令

激活流程：
  1. 设置U-Boot环境变量：
     - boot_part = B
     - boot_retry_count = 0
     - boot_commit_timeout = 120（秒）
  
  2. 设置回滚保护：
     - 写入回滚标记至配置分区
     - 记录当前版本号用于回滚
  
  3. 执行系统重启：
     - 调用 reboot 系统调用
     - 等待U-Boot引导至新分区

退出动作：
  - 无（系统重启）
```

### 4.7 COMMITTING 状态

```
进入动作：
  - 上报状态：COMMITTING
  - 启动健康检查定时器（120秒窗口）

健康检查项：
  1. 系统服务检查：
     - 核心进程（OTA Daemon、导航、感知）存活
     - CPU占用率 < 80%
     - 内存占用 < 阈值
  
  2. 硬件通信检查：
     - 各MCU节点通信正常
     - 传感器数据正常上报
  
  3. 功能冒烟测试：
     - 电机自检通过
     - 传感器数据采集正常

COMMIT动作（健康检查通过）：
  - 设置U-Boot环境变量：boot_commit = 1
  - 标记当前分区为稳定版本
  - 上报升级成功

超时处理（120秒内未COMMIT）：
  - 自动触发回滚

退出动作：
  - 上报COMMIT结果
```

### 4.8 ROLLBACK 状态

```
进入动作：
  - 上报状态：ROLLBACK
  - 记录回滚原因

回滚流程：
  1. 确定回滚目标分区：
     - 读取配置分区中记录的上一个稳定版本分区
  
  2. 设置U-Boot环境变量：
     - boot_part = 旧分区
     - boot_rollback = 1
  
  3. 执行系统重启：
     - 调用 reboot
     - 引导至旧分区
  
  4. 回滚后处理：
     - 上报回滚完成事件
     - 云端记录回滚原因
     - 暂停该版本后续推送

退出动作：
  - 上报回滚完成
```

### 4.9 ERROR 状态

```
进入动作：
  - 上报状态：ERROR
  - 上报错误码和错误详情
  - 停止所有升级相关操作

错误恢复：
  - 等待云端下发错误清除指令
  - 或等待新的升级指令（云端重新推送）

退出动作：
  - 清除错误标记
  - 重置状态机至IDLE
```

## 5. 状态机实现约束

### 5.1 状态持久化
- 当前状态必须持久化至配置分区，防止断电后状态丢失
- 状态变更时先持久化，再执行动作

### 5.2 状态上报
- 每次状态变更必须通过MQTT上报云端
- 上报内容：设备ID、状态码、时间戳、附加信息（进度/错误码）

### 5.3 超时保护
- 每个状态设置最大停留时间，超时触发异常处理

| 状态 | 超时时间 | 超时处理 |
|------|---------|---------|
| PRECHECK | 30s | 上报超时，回退IDLE |
| DOWNLOADING | 30min | 上报超时，进入ERROR |
| VERIFYING | 60s | 上报超时，进入ERROR |
| FLASHING | 10min | 上报超时，进入ERROR |
| COMMITTING | 120s | 触发回滚 |

### 5.4 并发控制
- 状态机为单实例，同一时刻只能处于一个状态
- 升级过程中拒绝新的升级指令（除非取消/回滚指令）

## 6. 错误码定义

| 错误码 | 含义 | 触发状态 |
|--------|------|---------|
| 0x1001 | 存储空间不足 | PRECHECK |
| 0x1002 | 电量不足 | PRECHECK |
| 0x1003 | 网络质量差 | PRECHECK |
| 0x1004 | 版本不兼容 | PRECHECK |
| 0x2001 | 下载超时 | DOWNLOADING |
| 0x2002 | 下载重试耗尽 | DOWNLOADING |
| 0x3001 | SHA256校验失败 | VERIFYING |
| 0x3002 | 签名校验失败 | VERIFYING |
| 0x3003 | 版本信息不匹配 | VERIFYING |
| 0x4001 | Flash写入失败 | FLASHING |
| 0x4002 | 差分还原失败 | FLASHING |
| 0x5001 | 健康检查失败 | COMMITTING |
| 0x5002 | COMMIT超时 | COMMITTING |
| 0x6001 | 回滚失败 | ROLLBACK |
| 0xFFFF | 未知错误 | 任意状态 |
