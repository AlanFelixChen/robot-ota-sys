# OTA系统设计文档

## 1. 概述

### 1.1 系统目标
为机器人端Linux主控及多MCU节点提供安全可靠的远程固件升级能力，支持全量/差分升级、A/B分区无缝切换、升级失败自动回滚。

### 1.2 设计原则
- 安全优先：全链路签名校验，硬件信任根，传输加密
- 断点续传：支持下载中断后从断点恢复，避免重复下载
- 原子切换：A/B分区原子切换，升级失败不影响当前运行系统
- 多MCU协同：统一编排Linux主控与多MCU节点的升级顺序

## 2. 核心流程设计

### 2.1 升级主流程

```
IDLE → PRECHECK → DOWNLOADING → VERIFYING → FLASHING → ACTIVATING → COMMITTING → IDLE
  ↑                                                                    │
  └────────────────── ROLLBACK ←──────────────────────────────────────┘
```

| 阶段 | 动作 | 说明 |
|------|------|------|
| PRECHECK | 存储空间、电池电量、网络质量检查 | 任一条件不满足则中止升级 |
| DOWNLOADING | 从云端/边缘网关下载固件包 | 支持断点续传，Range请求 |
| VERIFYING | ECDSA签名校验 + SHA256完整性校验 | 校验失败立即中止 |
| FLASHING | 将固件写入备用分区（App B） | 差分包先还原再写入 |
| ACTIVATING | 设置U-Boot启动标记，重启设备 | 引导至新分区 |
| COMMITTING | 健康检查通过后确认升级 | 标记新分区为稳定版本 |
| ROLLBACK | 健康检查失败或手动触发 | 回退至旧分区 |

### 2.2 回滚流程

触发条件：
- 新系统启动后健康检查失败（CPU/内存/核心进程异常）
- 预设时间窗口内未收到COMMIT确认
- 云端主动下发回滚指令

执行步骤：
1. U-Boot检测启动失败计数超过阈值
2. 自动切换启动分区至旧版本
3. 设备端上报回滚事件至云端
4. 云端记录回滚原因，暂停该版本推送

### 2.3 MCU协同升级流程

```
Linux主控下载固件包 → 解析MCU固件 → 逐MCU发起升级
                                    ├── 电源MCU：UART进入Bootloader → 分块传输 → 校验 → 重启
                                    ├── 传感器MCU：UART进入Bootloader → 分块传输 → 校验 → 重启
                                    ├── 电机MCU：CAN进入Bootloader → 分块传输 → 校验 → 重启
                                    └── 灵巧手MCU：UART进入Bootloader → 分块传输 → 校验 → 重启
```

- 多MCU可并行升级，最大并发数由DAG编排器控制
- 任一MCU升级失败，整体升级标记为失败，支持单MCU重试

## 3. 关键决策记录

| 决策项 | 方案 | 理由 |
|--------|------|------|
| 升级引擎 | SWUpdate | 原生支持A/B分区、差分升级、签名校验，社区活跃 |
| 差分算法 | zchunk | 压缩率高（10%-30%），支持流式解压，适合嵌入式 |
| 分区布局 | A/B + 数据 + 配置 | 原子切换，回滚简单，数据与系统隔离 |
| 文件系统 | UBIFS | Raw Flash原生支持，掉电保护，坏块管理 |
| 安全芯片 | ATECC608 | 硬件信任根，私钥不可导出，ECC加速 |
| 通信协议 | MQTT + HTTPS | MQTT长连接低延迟指令，HTTPS大文件断点续传 |

## 4. 异常处理矩阵

| 异常场景 | 处理策略 |
|---------|---------|
| 下载中断 | 断点续传，最多重试3次 |
| 存储空间不足 | PRECHECK阶段检测，提示清理空间 |
| 电量不足 | PRECHECK阶段检测，要求电量>30%或接入充电 |
| 签名校验失败 | 立即中止，上报安全事件 |
| Flash写入失败 | 重试写入，超过3次触发回滚 |
| 新系统启动失败 | U-Boot自动回退旧分区 |
| MCU升级超时 | 单MCU重试，不影响其他MCU
